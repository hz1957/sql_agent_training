Proposal: Execution-Guided RL for Text-to-SQL Agents

Train a SQL agent through interaction with a database environment: propose a query, observe execution feedback, revise when needed, and optimize the policy toward correct final results in a CUDA training setup.

Idea: Interactive Agent Loop
Signal: Execution Feedback
Learning: Reinforcement Learning
Compute: CUDA Environment

Research Idea

Motivation and Hypothesis

  • Problem: Text-to-SQL models are often trained to imitate gold SQL, but real task success depends on whether the query actually executes and returns the right result.
  • Idea: Treat the database as an interactive environment rather than a passive label source.
  • Hypothesis: RL over execution-grounded trajectories can improve correctness, self-repair, and schema grounding beyond an SFT-only policy.
  • Why now: Strong code LMs plus GPU training make it practical to test agent-level learning instead of only prompt engineering.
Claim to test: a SQL agent should be optimized on complete interaction trajectories, not just one-shot SQL strings.

Agent-Centric Method

The Agent Loop Is the Training Unit

The full write-check-rewrite trajectory becomes the RL training signal; success exits the loop and finalizes the answer.

Evaluation Plan

Metrics, Baselines, and Risks

Exec Acc Primary metric: final SQL returns the same result as the reference query on held-out databases.
Repair Rate How often the agent turns an initially failing query into a correct final answer after feedback.
Generalization Performance across unseen schemas, harder joins, nested queries, and aggregation-heavy questions.
RL Stability Reward trend, KL drift, invalid SQL rate, and execution failure rate during training.
Baselines Compare SFT-only and prompt-only agents against an RL-trained interactive agent.
Models Use capable code-oriented LMs in the CUDA environment; scale model size with available GPU budget.
Risks Sparse rewards, execution-equivalent SQL, overfitting to feedback patterns, and unsafe query behavior.
Expected contribution: a proposal for turning text-to-SQL from one-shot generation into an execution-grounded RL agent problem.
